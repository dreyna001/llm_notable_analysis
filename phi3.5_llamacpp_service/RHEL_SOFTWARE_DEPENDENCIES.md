# RHEL-class host software (`phi3.5_llamacpp_service`)

Use this as a checklist for a **fresh minimal** RHEL, Rocky Linux, or AlmaLinux image **before** running `install_phi35_nonroot_offline.sh`. The non-root installer does not install RPMs for you; apply this checklist (or your SOE equivalent) first. The offline flow is summarized as step **0** in [`OFFLINE_PRESTAGE_GUIDE.md`](OFFLINE_PRESTAGE_GUIDE.md). Substitute `dnf` with `microdnf` or `yum` if your image uses those.

If you prefer a single sudo entrypoint, `install_phi35_sudo.sh` can install these same packages first (`PHI_INSTALL_PACKAGES=true` by default), then invoke the non-root installer.

Artifacts (llama.cpp source + GGUF) are **not** covered here; see `OFFLINE_PRESTAGE_GUIDE.md`.

---

## 1) Packages the install script explicitly requires

The script runs `command -v` for **`cmake`**, **`make`**, **`curl`**, and **`sha256sum`** before it does anything else. On RHEL, `sha256sum` comes from **coreutils**.

| Role | Typical RPM packages |
|------|----------------------|
| Shell | `bash` (BaseOS; usually present) |
| Checksums / basic file utils (`sha256sum`, `install`, etc.) | `coreutils` |
| Local HTTP health probe | `curl` |
| Configure llama.cpp | `cmake` |
| Build (Makefiles generated by CMake) | `make` |
| C/C++ toolchain for compiling llama.cpp | `gcc`, `gcc-c++` |

**Typical one-liner (online registrar / build host):**

```bash
sudo dnf install -y bash coreutils curl cmake make gcc gcc-c++
```

**Airgapped / offline install:** mirror the same RPM set (and their dependencies) into a local repo or install from copied RPMs; `dnf install ./path/*.rpm` with dependencies resolved offline.

---

## 2) CMake version

Pinned `llama.cpp` in this package expects a **reasonably current** CMake (upstream usually requires **3.14+**; newer tags may raise the floor). RHEL 8 / 9 AppStream `cmake` is normally sufficient. If `cmake` is missing or too old, use your org’s approved **developer toolset** or **cmake3** pattern consistent with your SOE.

---

## 3) Optional packages

| Package | When you need it |
|---------|------------------|
| `git` | Only if your pre-staged `llama.cpp` tree still contains a `.git` directory and you want the installer’s **non-fatal** commit warning vs `PHI_EXPECTED_RUNTIME_COMMIT`. Not required for build or run. |

---

## 4) If configure/build fails (common add-ons)

These are **not** hard-coded in the installer; add them only if CMake or the compiler reports missing headers or tools:

| Symptom / hint | Typical RPM packages |
|----------------|----------------------|
| SSL/TLS-related CMake errors | `openssl-devel` |
| `pkg-config` / `.pc` discovery issues | `pkgconf-pkg-config` |
| zlib-related missing symbols/headers | `zlib-devel` |

---

## 5) Runtime (after a successful build)

The built `llama-server` binary is dynamically linked. On the same machine that compiled it, shared libraries are usually already satisfied. For **deploying only the binary** to another host, run on that host:

```bash
ldd "$HOME/.local/bin/llama-server"
```

and ensure every listed `.so` is provided by installed RPMs (commonly `glibc`, `libstdc++`, `libgcc`, `libgomp` if OpenMP is linked, `libm`, etc.).

---

## 6) `PHI_SKIP_RUNTIME_BUILD=true` caveat

The script **still** checks for **`cmake`** and **`make`** even when skipping the compile step. If you only ship a prebuilt `llama-server`, you either keep minimal `cmake` + `make` installed or adjust the script to skip those checks (not done in the stock script).

---

## 7) Capacity (not RPMs, but required in practice)

- **Disk:** GGUF baseline is on the order of **~2.3 GiB**; the CMake build tree is often **multiple GiB**.
- **RAM:** enough to **mmap/load** the model for inference (significantly above the file size is typical once KV cache and overhead are included; tune `PHI_CTX_SIZE` / concurrency to your host).

---

## 8) What this path does **not** pull in

- **No GPU / CUDA stack** in the default installer flags.
- **No root services** (no systemd unit in this script).
- **No Python / Node / container runtime** for this workflow.
