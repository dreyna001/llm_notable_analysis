# Customer Onboarding Template

## Purpose

Capture customer-specific facts, mappings, routing, validation expectations, and handoff requirements without changing reusable product contracts.

This template should eventually help record:

- customer summary
- selected application state
- source definitions and mappings
- known data quirks
- output and handoff requirements
- validation and signoff expectations

## Current State

This file exists to complete the standard document-stack layout.

It is intentionally generic and should not be used to redefine the shared product model.

## One-Line Summary

Use this file later to capture customer-specific deployment and data details without hard-coding them into core product docs.
