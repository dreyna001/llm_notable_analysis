# Governance / Compliance Artifact Templates (Drafts)

These documents are **templates** to help prepare for a formal security assessment/authorization workflow (e.g., NIST RMF/ATO or “FedRAMP-equivalent controls” in an on‑prem enclave).

## Important

- These templates are **not** an authorization and do **not** claim compliance.
- Several artifacts are typically **owned or finalized by external roles**:
  - **SAP**: usually owned/finalized by the assessment team/3PAO
  - **SAR**: produced by the assessment team after testing
  - **ATO letter**: issued by the Authorizing Official (AO)
- Use bracketed placeholders like `[TBD]` and `[ORG PROVIDED]` where information must come from the customer environment, enclave, or assessor.

## Files

- `SAP_TEMPLATE.md`: Draft Security Assessment Plan template.
- `SAR_TEMPLATE.md`: Security Assessment Report template (assessor-produced; provided for structure).
- `POAM_TEMPLATE.md`: POA&M tracker template (findings/remediation tracking).
- `INHERITED_CONTROLS_MATRIX_TEMPLATE.md`: Inheritance matrix template (customer/enclave inputs required).
- `ATO_LETTER_TEMPLATE.md`: Authorization Decision / ATO letter template (AO-only).
